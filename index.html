fmd Tool: File Metadata and Forensic Analysis ▼

Overview: fmd, short for File Metadata, is a forensic metadata tool designed to extract useful information from files during malware analysis, threat hunting, and digital forensics work.

Supported File Types

  • Windows LNK files
  • Windows PE files
  • Mach-O binaries
  • ELF binaries

Key Capabilities

  • Extracts file metadata for forensic review
  • Performs hashing, entropy checks, mime type detection, and string extraction
  • Supports recursive directory analysis with configurable depth
  • Filters files by extension or interesting mime types
  • Reports PE section information that may help identify packed binaries
  • Can collect additional NTFS-related artifacts, such as alternate data streams and certain timestamps, when run with elevated privileges

Example Usage

fmd --pretty --depth 3 --extensions 'exe,dll,pif,ps1,bat,com'
fmd --pretty --depth 3 --extensions 'not:exe,dll,pif,ps1,bat,com'

Malware Detection Project: Methodology, Challenges, and Integration with the fmd Tool ▼

Introduction: In this project, I developed machine learning models to detect malware using data from various sources, including API functions, DLLs imported, PE headers, and PE sections. The models were evaluated using Logistic Regression and Random Forest algorithms, with and without Principal Component Analysis (PCA). This section details the process followed, challenges encountered, and how the results can be integrated with the fmd tool.

Methodology

Data Preparation: The project started by aggregating and preprocessing data, ensuring no duplicate columns. The target variable 'Type' indicated whether the file was malicious. The top 500 features were selected using the SelectKBest method based on the ANOVA F-statistic, and SMOTE was used to balance the class distribution. Logistic Regression and Random Forest models were trained and evaluated with a 70-30 train-test split, with and without PCA, using GridSearchCV for hyperparameter tuning.

Model Performance

The Random Forest models outperformed Logistic Regression models in AUC-ROC, accuracy, precision, recall, and F1-score metrics. The Random Forest model without PCA had the highest AUC-ROC score of 0.9851 and an accuracy of 0.89.

Challenges Encountered

Convergence Issues: During training, the Logistic Regression models faced convergence warnings due to the lbfgs solver. Increasing the max_iter parameter mitigated but did not entirely resolve the issue.
Indexing Errors: An "index out of bounds" error occurred during data visualization, resolved by correct index matching.
Class Imbalance: SMOTE effectively addressed the dataset's class imbalance, leading to more reliable model performance.

Integration with the fmd Tool

The project's results can be integrated with the fmd tool for preprocessing, feature extraction, and real-time analysis. The fmd tool can extract metadata such as PE sections, mime types, and file sizes, which can be directly used in machine learning models for malware detection. This integration enhances the detection process, making it suitable for forensic and operational cybersecurity environments.

Conclusion

Integrating the results from this project with the fmd tool improves malware detection accuracy and provides richer context, making it a valuable addition to cybersecurity tools.
